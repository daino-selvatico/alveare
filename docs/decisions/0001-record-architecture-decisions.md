# 0001 — Record architecture decisions

- Status: accepted
- Date: 2026-06-19

## Context

This is a long, exploratory project with many forced technical choices (toolchain, quant format, kernel strategy, target model order). Future-us will not remember why.

## Decision

Record every significant, hard-to-reverse decision as a short ADR in `docs/decisions/`, numbered sequentially. Format: Context → Decision → Consequences. Keep them short.

## Consequences

- One more small file per decision.
- A readable trail of *why*, not just *what*, which matters for a research-y project where many things are tried and discarded.
